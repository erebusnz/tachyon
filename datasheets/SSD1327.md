## Description

The SSD1327 is a 128x128, 4-bit grayscale OLED/PLED display controller from Solomon Systech. It contains a 128x128x4-bit GDDRAM (8192 bytes), 16-level grayscale lookup table, 128-channel segment driver, and 128-channel common driver. The controller supports 8080/6800 parallel, 3-wire SPI, 4-wire SPI, and I2C interfaces. Each pixel is represented by 4 bits (16 grayscale levels), and two adjacent pixels are packed into a single byte in RAM. This is the OLED controller used in the Tachyon module, providing 4-bit grayscale for rich UI rendering of waveforms, menus, and parameter displays over SPI2.

The BS0 pin selects the interface mode: tie LOW (or 0-ohm to GND) for 4-wire SPI, tie HIGH for I2C. The module ships with BS0 = 0 (4-wire SPI) by default.

## Application Notes

- **SPI interface:** The SSD1327 in 4-wire SPI mode uses SCLK, SDIN, CS#, and D/C#. Data is clocked in on the rising edge of SCLK, MSB first. The SPI mode is CPOL = 0, CPHA = 0 (SPI Mode 0). CS# must be held low during the entire transaction. The D/C# pin selects between command bytes (LOW) and data bytes (HIGH), and must be set before CS# goes low. Maximum SCLK is 10 MHz. The OLED runs on a dedicated SPI1 bus at PA5 (SCK), PA7 (MOSI), PB12 (CS), PB4 (DC), PB5 (RES) — separate from the DAC8552 on SPI2, avoiding SPI mode conflicts and bus contention during DMA transfers.
- **Initialization sequence:** After hardware reset (RES# low for at least 3 us, then high, then wait at least 3 us), the SSD1327 requires a series of configuration commands: set display off (0xAE), set remap/dual COM (0xA0), set display start line (0xA1), set display offset (0xA2), set multiplex ratio (0xA8, 0x7F for 128 rows), set function selection A (0xAB), set phase length (0xB1), set front clock divider/oscillator (0xB3), set GPIO (0xB5), set second pre-charge period (0xB6), set VCOMH voltage (0xBE), set pre-charge voltage (0xBC), set function selection B (0xD5), set command lock (0xFD), then set display on (0xAF). A typical initialization can be adapted from Waveshare example code.
- **Framebuffer size:** Each byte of GDDRAM holds two pixels (upper nibble and lower nibble). The framebuffer is 8192 bytes (128x128/2). Full-screen DMA transfers over SPI2 at 10 MHz take approximately 6.6 ms. At a 10-30 Hz UI refresh rate, this leaves ample SPI2 bandwidth for interleaved DAC8552 updates.
- **Grayscale lookup table:** The SSD1327 has a programmable 16-entry grayscale lookup table (command 0xB8) that maps the 4-bit pixel value to a pulse width (current drive duration). This allows gamma correction or custom contrast curves. The default linear table can be restored with command 0xB9.
- **Column address format:** Because two pixels are packed per byte, the column address range is 0x00 to 0x3F (64 column addresses for 128 pixel columns). When writing data, set the column address window (command 0x15) from 0x00 to 0x3F, and the row address window (command 0x75) from 0x00 to 0x7F for a full-screen write. Data bytes are written left-to-right, top-to-bottom.
- **Remap configuration (0xA0):** The remap register controls column address mapping, nibble remap, horizontal/vertical address increment, COM remap, and COM split odd/even. For a typical 128x128 module with the connector at the top, a remap value of 0x51 (enable column remap, enable nibble remap, enable COM remap) is commonly used, but the exact value depends on panel orientation.
- **Power sequencing:** VDD (logic, 3.3 V) should be applied first, then VCC (panel drive, typically 12 V generated by an on-board boost converter on the Waveshare module). On power-down, turn off VCC first, then VDD. The display should be turned off (0xAE) before removing VCC. The Waveshare module handles the VCC boost internally, so only the 3.3 V rail needs to be supplied externally.
- **Partial display / scrolling:** The SSD1327 supports hardware scrolling (continuous horizontal scroll, commands 0x26/0x27) and partial display modes. Scrolling can be useful for marquee-style parameter names in the Eurorack UI without redrawing the framebuffer.
- **I2C alternative:** If SPI2 pins become constrained, the SSD1327 also supports I2C at up to 400 kHz. The I2C slave address is 0x3C (DC pin low) or 0x3D (DC pin high). However, I2C throughput is significantly lower than SPI for full-screen refreshes.
- **Contrast control (0x81):** Sets the segment output current level from 0 to 255. Higher values increase brightness. Default is 0x7F. This can be used to implement display dimming, for example based on a Eurorack panel brightness control or ambient light sensor.
