## Description

The DAC8552 is a 16-bit, dual-channel, voltage output digital-to-analog converter from Texas Instruments. It features an on-chip output buffer amplifier providing rail-to-rail output swing, a double-buffered input architecture, and an ultralow glitch energy of 0.15 nV-s. The device communicates via a 3-wire SPI-compatible serial interface supporting clock rates up to 30 MHz and powers on with outputs at zero-scale.

## Application Notes

- **SPI protocol:** The DAC8552 uses a 3-wire SPI-compatible interface (DIN, SCLK, SYNC). Data is clocked in on the falling edge of SCLK (CPOL = 0, CPHA = 1, i.e. SPI Mode 1 -- though the device also works with CPOL = 1, CPHA = 0, SPI Mode 2, since it simply clocks on the falling edge of SCLK). SYNC is active low and frames each 24-bit transaction.
- **24-bit shift register format:** Each write consists of 24 bits shifted in MSB first: bits [23:22] are unused (set to 0), bit [21] selects the DAC channel (0 = DAC A, 1 = DAC B), bit [20] is the buffer select (0 = write to buffer, 1 = write to buffer and load DAC), bits [19:18] are power-down mode select (00 = normal operation, 01 = 1 kohm to GND, 10 = 100 kohm to GND, 11 = high-Z), and bits [17:16] are unused. Bits [15:0] are the 16-bit DAC data word.
- **Power-on state:** On power-up, the internal registers are reset to zero and both DAC outputs are at zero-scale (0 V). The device powers up in normal operating mode (not powered down).
- **Decoupling:** Place a 1 uF to 10 uF ceramic bypass capacitor on VDD as close to the pin as possible. A 100 nF ceramic capacitor in parallel is recommended for high-frequency noise rejection. Decouple the VREF pin with a 1 uF to 10 uF ceramic capacitor to GND.
- **Reference input:** The VREF pin accepts a reference voltage from 0 V up to VDD. The output voltage of each channel is VOUT = VREF x (D / 65536), where D is the 16-bit code. For best performance, use a low-noise, low-drift precision voltage reference (such as the REF5025 also in this BOM). The reference input impedance is code-dependent; a reference buffer or low-impedance source is recommended.
